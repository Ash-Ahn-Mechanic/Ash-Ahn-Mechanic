# ROKEY 스터디 커리큘럼 (비전·백엔드 비필수)

## 스터디 정체성
AI·로봇 분야 실무 역량 강화를 목표로 하되, 특정 언어나 백엔드 기술에 종속되지 않는 프로젝트 중심 스터디

- 언어: 자유 (Python 권장, 강제 아님)
- 핵심: 문제 정의 → 구현 → 결과 → 설명
- 결과물: 실행 가능 프로젝트 + 문서

---

## 전체 목표
- 프로젝트형 사고 습득
- 협업을 고려한 코드·문서 구조화
- ROKEY 활동으로 설명 가능한 결과물 확보

---

## 1주차 — 프로젝트형 사고 & 협업 기초
**목표**
- 과제형 코드에서 프로젝트형 코드로 사고 전환
- 최소 협업 규칙과 프로젝트 구조 합의

**주요 내용**
- 프로젝트형 사고 개념
- 최소 프로젝트 폴더 구조
- GitHub 협업 최소 세트 (Repo, Issue, PR, README)

**참고**
- Kaggle (문제 정의 및 데이터 기반 사고 참고): https://www.kaggle.com
- HAI – Human-Centered AI (문제 정의·맥락화 관점): https://hai.stanford.edu

**산출물**
- 개인 미니 프로젝트
- README 포함 GitHub 업로드
